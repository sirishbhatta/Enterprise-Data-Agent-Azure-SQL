🏗️ Architecture Overview

The system uses a sophisticated multi-path query approach:

PATH A: Direct Query

Standard query execution through the selected database

PATH B: Follow-up Rewrite

Intelligent query rewriting for follow-up questions

PATH C: Memory Hit

Retrieval from vector memory cache for similar questions

PATH D: Model Cascade

Intelligent LLM fallback when primary model unavailable

PATH E: Federated Query

Parallel queries across multiple databases with result merging
