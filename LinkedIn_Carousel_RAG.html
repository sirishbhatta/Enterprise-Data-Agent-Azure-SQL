5 Query Paths: Federated Joins

5/5
PATH E: Federated Query (Cross-Database Merge)
Question needs data from 2+ databases. Queries run parallel, then merged in Python.
1
Routing recognizes federated
"Show HR employees earning $140k+ with their total claims"
2
Two queries generated
PostgreSQL: SELECT emp_id, name, salary FROM hr_employees WHERE salary > 140000
SQL Server: SELECT memberid, SUM(claims) FROM claims GROUP BY memberid
3
Parallel execution
Both databases queried simultaneously. Results cached.
4
Python merge (pandas)
final_df = merge on emp_id = memberid
Single unified table shown to user
Why not SQL JOIN? SQL can't natively join across different database servers (PostgreSQL on one machine, SQL Server on another). Python's pandas handles the merge in memory — this is the "federated" approach.
